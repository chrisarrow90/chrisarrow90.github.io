Differences between relative, absolute and fixed positioning

Relative Positioning

When an element is given the property position: relative; it remains positioned according to the normal flow of the document. Initially, nothing will happen unless an offset is applied. To apply an offset, you add the additional property of either top, right, bottom or left with the value representing the offset relative to itself (from it's normal position). The element position remains according to the normal flow of the document. Note: The property left does not move the element left, it is the offset value from the left edge (in other words, element will move right)

Absolute Positioning

When position: absolute; is applied to an element, it is removed from the normal document flow. It will be postioned relative to its closet positioned ancestor, if any, otherwise relative to the initial containing block (i.e the html)

In the example below, box two has been given an absolute position and offset values of 20px from top and 20px from left. The element is removed from the normal flow and the other elements are positioned as if it didn't exist. Since the box is a inside a <div> container with position: relative set on it, the offset value is relative to the edge of that container

Fixed Positioning

When position: fixed; is applied to an element, it is removed from the normal document flow and positioned relative to the initial containing block established by the viewport or browser window. This can be used to create a "floating" element that stays in the same position regardless of scrolling and is useful for navigation bars at the top of a page.

Summary

In summary only relative positioning keeps element in the normal flow. Using either absolute or fixed positioning removes the element from the normal flow and other elements will behave as if it didn't exist. You can then position the element how you want including making it overlap with other elements

The real difference between absolute and fixed is an element with an absolute position will positioned relative to it's containing block (nearest positioned ancestor), if there is no positioned ancestor, then the viewport will be the containing block. An element with a fixed position however, remains stationary in the viewport window in all cases and is not affected by scrolling. This is particularly useful for navigation bars.
